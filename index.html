gestor-rsl

API REST para gestionar la extraccion de 176 articulos cientificos de una Revision Sistematica de Literatura sobre incendios forestales, PM2.5 e IRA en Ancash, Peru.
Proyecto del curso Lenguajes de Programacion (1INF13) — Maestria en Informatica, PUCP.

Java 21 Spring Boot 3.5 Spring Data JPA MySQL Lombok Maven

Arquitectura en capas

El proyecto sigue el patron controller → service → repository → modelo, estandar de Spring Boot.

Modelo de datos

4 tablas relacionadas. ARTICULO es la tabla central con 8 campos bibliograficos. Las tablas de extraccion se vinculan mediante FK idArticulo.

Endpoints

API disponible en http://localhost:8080

GET/api/articulosListar (filtros: cadena, estado, pais, contaminante, q)
GET/api/articulos/{id}Obtener por id
GET/api/articulos/duplicadosArticulos con DOI duplicado
POST/api/articulosCrear articulo
POST/api/articulos/importar-doiImportar metadatos desde DOI (Crossref API)
PUT/api/articulos/{id}Actualizar articulo
DELETE/api/articulos/{id}Eliminar articulo
GET/api/dashboardProgreso por cadena de busqueda
GET/api/reportes/frecuencias?campo=contaminantesFrecuencias de contaminantes, metodos, software o paises
GET/api/reportes/lagsDistribucion de lags exposicion → IRA
GET/api/reportes/matrizMatriz pais x topografia
